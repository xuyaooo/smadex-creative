# Creative.AI — Your AI Creative Companion

A full-stack creative-performance prediction system for mobile advertising creatives. Drop an ad, get a Health Score, a structured analysis, an AI rebuild, and live coaching. Built on the synthetic Smadex Creative Intelligence Challenge dataset (1,076 creatives × 36 advertisers × 6 verticals).

## What it does

Drop a screenshot of any ad creative. Behind the scenes the app chains **three personalized models**:

1. **Soft-voting tabular ensemble** — XGBoost 5-seed bag + LightGBM + CatBoost + HistGBM + LogReg → 4-class status (top performer / stable / fatigued / underperformer) + a 0–100 health score + counterfactual lift suggestions.
2. **Personalized VLM** — SmolVLM-Instruct, fully fine-tuned with self-distillation on Gemma teacher labels. Reads the creative + the ensemble's diagnosis and emits structured JSON: strengths, weaknesses, fatigue risk, top recommendation, color/layout/copy fixes with causal "why" reasoning.
3. **Flux edit (image rebuild)** — Rank-32 LoRA + reward-weighted DPO fine-tune of Flux edit, trained on synthetic positive samples generated by Nano Banana under the tabular ensemble's lift brief.

Every result page also shows:
- A **14-day lifecycle forecast** (predicted CTR / impressions / ROAS curves) from the trained `train_lifecycle.py` model
- A **data-grounded color palette** extracted via k-means from real top-performer images in the user's vertical
- A **draw mode** with a "live coach" — circle any element, the coach (Maya) tells you if it's working or how to fix it, the circle auto-erases

## Quickstart

```bash
# 1. Install once (~2 minutes)
pip install -r models/requirements.txt
(cd front && npm install)

# 2. Set the OpenRouter key (drives the live VLM analysis + image edit)
echo "VITE_OPENROUTER_API_KEY=sk-or-..." > front/.env.local

# 3. Boot everything
./run.sh
```

- **Frontend** → http://localhost:5173
- **Backend**  → http://localhost:8000

The frontend is self-contained — it ships with precomputed predictions for every creative, so it renders without the backend running. The OpenRouter key is only needed for the *live* AI features (metadata extraction from a fresh image, the analysis JSON, the AI image edit).

## Reproduce from raw CSVs

```bash
# Tabular ensemble (Model 1) — ~17s on CPU
python3 models/scripts/build_clean_dataset.py
python3 models/scripts/train_clean.py --final

# Lifecycle curves + per-vertical palettes (front-end lookup tables)
PYTHONPATH=models python3 models/scripts/train_lifecycle.py
PYTHONPATH=models python3 models/scripts/build_palette_lookup.py

# Personalized SmolVLM (Model 2) — full fine-tune, ~45 min on a single H100
export OPENROUTER_API_KEY=sk-or-...
PYTHONPATH=models python3 models/scripts/label_with_openrouter.py
PYTHONPATH=models python3 models/scripts/finetune_smolvlm_full.py \
    --epochs 3 --batch 4 --grad_accum 8 --lr 5e-6

# Flux edit (Model 3) — generate + train, ~6h on a single H100
PYTHONPATH=models python3 models/scripts/generate_flux_pairs.py --max 200 --threshold 75
PYTHONPATH=models python3 models/scripts/finetune_flux_edit.py \
    --epochs 4 --batch 1 --grad_accum 16 --rank 32 --dpo
```

End-to-end Model 1 from raw CSVs → trained ensemble: **~4 min on CPU**. Models 2 + 3 require GPU and OpenRouter credits.

## Honest caveats

- The dataset is **synthetic**. A predict-by-vertical-prior baseline gets ~0.30 macro-F1 for free; the ensemble adds +0.35 on top, mostly via early-life CTR aggregates.
- Top-performer F1 has a 95% bootstrap CI of [0.31, 0.89] (n=11 in test). Point estimate is directional.
- Pause/Pivot recommendation precision ≈ 0.54 — surface it for human review, don't autopilot.
- Lifecycle curve XGBoost regressor has r² ≈ 0.07 on shape; we ship the **bucket-means** of real curves, not the regressor's per-sample prediction, because the bucket means are more honest.
- Cold-start: the 7-day early-life signal is the strongest feature. For brand-new creatives with zero impressions the model degrades to the visual-rubric + metadata baseline (~0.45 macro-F1).
